## Table name - locations

# Locations Table Schema Summary

## Columns

| Name            | Type         | Description                        | Example Value   |
|-----------------|--------------|------------------------------------|-----------------|
| location_id     | INTEGER      | Unique identifier for each location| 1               |
| name            | VARCHAR(50)  | Name of the location               | Main Warehouse  |
| type            | VARCHAR(20)  | Type of location (e.g., Warehouse, Zone, Rack) | Warehouse      |
| parent_location | INTEGER      | ID of the parent location, allows hierarchical nesting | 1               |

## Keys

- **Primary Keys**
  - `location_id`: Uniquely identifies each row in the table.

- **Foreign Keys**
  - `parent_location`: References `location_id` in the same `locations` table to establish a hierarchy.

## Table name - sales_order_lines

# Table: sales_order_lines

## Columns

| Name              | Type           | Description                                    | Example Value |
|-------------------|----------------|------------------------------------------------|---------------|
| soline_id         | INTEGER        | Unique identifier for the sales order line     | 1             |
| so_id             | INTEGER        | Identifier for the related sales order         | 1             |
| product_id        | INTEGER        | Identifier for the product in the order line   | 1             |
| quantity_ordered  | INTEGER        | Number of units ordered for a particular line  | 5             |
| unit_price        | NUMERIC(10, 2) | Price per unit of the product ordered          | 95.00         |

## Keys

- **Primary Key**
  - `soline_id`: Uniquely identifies each order line within the table.

- **Foreign Keys**
  - `product_id`: References `product_id` in the `products` table, ensuring the product exists.
  - `so_id`: References `so_id` in the `sales_orders` table, linking the line to an order.
